# Implementation Plan

- [x] 1. エンティティ定義とレジストリの構築
- [x] 1.1 (P) エンティティ定義の値オブジェクトのテストと実装
  - エンティティ種別ID（正の整数）、表示名、フットプリントサイズ（Vector2i）を保持する不変の値オブジェクトを作成する
  - ID=0は予約値として拒否、空の表示名を拒否、フットプリントの幅・高さが正の整数であることを検証する
  - テストケース: 有効な引数での生成成功、ID=0での生成失敗、空の表示名での生成失敗、フットプリント0以下での生成失敗
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 7.5_

- [x] 1.2 (P) エンティティレジストリのテストと実装
  - エンティティ定義の登録・検索を一元管理するカタログを作成する（ItemCatalogパターンに準拠）
  - 同一IDの重複登録を拒否し、存在しないIDでの検索はnullを返す
  - MVPデフォルトデータ（Miner:1x1, Smelter:1x1, Belt:1x1, DeliveryBox:1x1）を登録する静的ファクトリメソッドを提供する
  - テストケース: 登録と検索の往復、重複登録の拒否、存在しないIDの検索でnull、デフォルトファクトリで4種全てが正しいフットプリントで登録されること
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 7.5_

- [x] 2. 配置済みエンティティの状態管理
- [x] 2.1 配置済みエンティティの値オブジェクトのテストと実装
  - 配置済みエンティティのメタデータ（entity_id, entity_type_id, 基準セル, 回転方向, フットプリント）を保持する値オブジェクトを作成する
  - 回転方向はEnums.Direction（N=0, E=1, S=2, W=3）を使用する
  - テストケース: 各フィールドの正しい保持、2x2エンティティと1x1エンティティそれぞれの生成
  - _Requirements: 1.3, 1.4, 3.3, 4.2_

- [x] 3. 配置システムのコアロジック
- [x] 3.1 配置可否判定のテストと実装
  - エンティティ種別IDと基準セルを指定して配置可能かどうかを判定するクエリメソッドを実装する
  - エンティティレジストリからフットプリントを取得し、CoreGridの範囲チェックと占有チェックを組み合わせて判定する
  - テストケース: 空きセルでの配置可能判定、占有済みセルでの配置不可判定、範囲外での配置不可判定、テスト専用2x2エンティティが境界をまたぐ場合の拒否、基準セル(63,63)への1x1エンティティ配置が空きの場合に許可される、基準セル(0,0)での1x1配置の許可
  - _Requirements: 2.1, 2.2, 2.4, 2.5_

- [x] 3.2 エンティティ配置のテストと実装
  - エンティティ種別ID、基準セル、回転方向を指定してエンティティを配置する機能を実装する
  - entity_idの自動採番、CoreGridへの占有登録（occupy_rect）、PlacedEntityの記録を一括で実行する
  - 配置失敗時はグリッド状態を一切変更しない
  - テストケース: 1x1エンティティの配置成功と占有状態の検証、テスト専用2x2エンティティの配置成功と4セルすべての占有検証、占有済みセルへの配置拒否とグリッド状態の不変性、範囲外への配置拒否とグリッド状態の不変性、配置即座完了（戻り値の即座の返却）
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2, 2.3, 6.1, 6.3_

- [x] 3.3 エンティティ撤去のテストと実装
  - セル座標を指定して配置済みエンティティを撤去する機能を実装する
  - CoreGridから該当セルのentity_idを取得し、PlacedEntityの基準セルとフットプリントに基づきvacate_rectで一括解放する
  - テスト専用2x2エンティティの場合、いずれか1セルの指定で全体を撤去する
  - テストケース: 1x1エンティティの撤去と占有解放の検証、テスト専用2x2エンティティの任意セル指定での全体撤去、撤去後の全占有セルの解放検証、空セル指定での撤去操作の無視、撤去即座完了、配置→撤去→同位置への再配置が成功すること
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 6.2_

- [x] 3.4 回転ロジックのテストと実装
  - 回転方向の切り替えロジック（N→E→S→W→N の時計回りサイクル）を実装する
  - 配置時に指定された回転方向がPlacedEntityに保持されることを検証する
  - 回転は占有領域を変化させない（MVPのエンティティはすべて正方形フットプリント）
  - テストケース: N→E→S→W→Nの完全サイクル、各方向での配置とPlacedEntityへの方向保持、回転前後で占有セルが変化しないこと
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 4. ゴーストプレビューの実装
- [x] 4.1 ゴーストプレビューの描画ノードの構築
  - 配置予定位置にフットプリントに応じたゴースト（半透明）スプライトを描画するノードを作成する
  - 配置可能時は緑色、不可能時は赤色でゴーストを表示する（PlacementSystem.can_place()を使用）
  - 対象セルの変更時にゴースト表示位置を即座に更新する
  - 選択中エンティティがない場合はゴーストを非表示にする
  - - [x] 4.1 Screenshot checkpoint: ゴーストプレビューが配置可否に応じて緑/赤で正しく表示されること
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 4.2 配置入力ハンドラの構築
  - プレイヤーの入力（配置、撤去、回転）をPlacementSystemのメソッド呼び出しに変換するノードを作成する
  - マウス位置からグリッドセル座標への変換を行う
  - 配置・撤去の結果に応じてゴーストプレビューを更新する
  - - [x] 4.2 Screenshot checkpoint: 配置・撤去操作が入力に応じて正しく動作すること
  - _Requirements: 1.1, 4.1, 5.4_

- [x] 5. 統合と検証
- [x] 5.1 システム統合
  - PlacementSystemをゲームシーンに統合し、CoreGridとEntityRegistryを接続する
  - 配置入力ハンドラとゴーストプレビューノードをシーンツリーに組み込む
  - 複数エンティティの連続配置→撤去→再配置のワークフローが正しく動作することを確認する
  - _Requirements: 1.1, 1.2, 4.1, 4.3, 6.1, 6.2_

- [x] 5.2 ヒューマンレビュー
  - - [x] 5.2 Human review: ゴーストの半透明度が視覚的に適切であること（背景グリッドと区別可能で、配置予定位置が一目で分かること）
  - - [x] 5.2 Human review: ゴースト表示がマウス移動に体感遅延なく追従すること
  - - [x] 5.2 Human review: 配置・撤去操作のレスポンスと操作感が直感的であること
  - _Requirements: 5.1, 5.2, 5.3, 5.4_
