# Implementation Plan

- [x] 1. ItemDefinition — アイテム種別の定義情報を保持する値オブジェクト
- [x] 1.1 ItemDefinitionのユニットテストを作成する
  - 正の整数IDと空でない表示名でインスタンスを生成し、各属性が正しく保持されることを検証する
  - ID=0での生成がアサーション失敗することを検証する
  - 空文字列の表示名での生成がアサーション失敗することを検証する
  - 生成後にIDと表示名が変更されないことを検証する
  - RefCountedを継承しシーンツリーなしでインスタンス化できることを検証する
  - _Requirements: 1.1, 1.2, 1.3, 6.1, 6.3_

- [x] 1.2 ItemDefinitionを実装する
  - RefCountedを継承した値オブジェクトとして、一意な正の整数IDと空でない表示名を保持する
  - コンストラクタでID > 0およびdisplay_name非空のアサーションを行う
  - ID=0は予約値として扱い、有効なアイテム定義に割り当てない
  - 生成後にIDと表示名は不変の属性として保持する
  - 1.1のテストがすべてパスすることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 6.3_

- [x] 2. ItemQuantity — アイテム数量の安全な増減と範囲クランプ
- [x] 2.1 (P) ItemQuantityのユニットテストを作成する
  - デフォルト値（初期値0、上限999）でインスタンスを生成し、初期状態が正しいことを検証する
  - カスタム初期値と上限値でインスタンスを生成し、各属性が正しく保持されることを検証する
  - 加算で数量が正しく増加することを検証する
  - 加算の結果が上限値を超える場合、上限値にクランプされることを検証する
  - 減算で数量が正しく減少することを検証する
  - 減算の結果が0を下回る場合、0にクランプされることを検証する
  - RefCountedを継承しシーンツリーなしでインスタンス化できることを検証する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 6.1_

- [x] 2.2 (P) ItemQuantityを実装する
  - RefCountedを継承し、数量値を0以上かつ上限値以下の有効範囲内に保持する
  - デフォルト上限999とし、生成時に任意の上限値を指定できるようにする
  - 加算メソッドで数量を増加し、上限を超えた場合はクランプする（エラーを発生させない）
  - 減算メソッドで数量を減少し、0を下回った場合はクランプする（エラーを発生させない）
  - 2.1のテストがすべてパスすることを確認する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_

- [x] 3. ItemCatalog — アイテム定義の登録・検索を一元管理するカタログ
- [x] 3.1 ItemCatalogのユニットテストを作成する（登録・検索）
  - 新しいアイテム定義を登録し、カタログに追加されることを検証する
  - 同一IDのアイテム定義を重複登録しようとした場合、登録が拒否されることを検証する
  - 重複登録の試行後、既存の登録が変更されていないことを検証する
  - 有効なIDで検索し、対応するアイテム定義が返されることを検証する
  - 存在しないIDで検索した場合、nullが返されることを検証する
  - ID=0で検索した場合、nullが返されることを検証する
  - 新しいアイテム定義の追加登録後、既存のアイテム定義が変更されていないことを検証する
  - RefCountedを継承しシーンツリーなしでインスタンス化できることを検証する
  - _Requirements: 2.1, 2.2, 2.3, 3.1, 3.2, 3.3, 6.1, 6.2_

- [x] 3.2 ItemCatalogを実装する（登録・検索）
  - RefCountedを継承し、Dictionaryを使用してID→ItemDefinitionのマッピングを管理する
  - アイテム定義の登録機能を実装し、同一IDの重複登録を拒否する
  - IDによるアイテム定義の検索機能を実装し、存在しないIDやID=0ではnullを返す
  - 登録済みアイテム定義の数を返す機能を提供する
  - 3.1のテストがすべてパスすることを確認する
  - _Requirements: 2.1, 2.2, 2.3, 3.1, 3.2, 3.3, 6.1, 6.2_

- [x] 3.3 MVP初期データのユニットテストを作成する
  - デフォルトカタログ生成で鉄鉱石（ID=1）が登録されていることを検証する
  - デフォルトカタログ生成で精錬鉄（ID=2）が登録されていることを検証する
  - ID=1で検索し鉄鉱石のアイテム定義が返されることを検証する
  - ID=2で検索し精錬鉄のアイテム定義が返されることを検証する
  - デフォルトカタログに2件のアイテム定義が登録されていることを検証する
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 3.4 MVP初期データのファクトリメソッドを実装する
  - 鉄鉱石（ID=1）と精錬鉄（ID=2）の2種が登録済みのカタログを生成するstaticファクトリメソッドを実装する
  - 3.3のテストがすべてパスすることを確認する
