# Implementation Plan

## Tasks

- [x] 1. ベルトタイルのデータモデル構築
- [x] 1.1 BeltTileDataの単体テスト作成
  - 方向（N/E/S/W）、アイテムID、進行度の初期値が正しいことを検証
  - アイテムの設定・クリア操作が正しく動作することを検証
  - 進行度の範囲（0.0〜1.0）が保持されることを検証
  - 下流接続情報（downstream_pos, has_downstream）の設定・取得を検証
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 1.2 BeltTileDataの値オブジェクト実装
  - 方向、アイテムID、進行度、下流接続情報を保持する値オブジェクトを実装
  - RefCountedベース、SceneTree非依存
  - 1タイルあたり最大1アイテムの制約を型レベルで表現（item_id: int、0=空）
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 2. ベルトグリッドのデータストア構築
- [x] 2.1 BeltGridの単体テスト作成
  - タイルの追加・削除・取得・存在チェックを検証
  - アイテムの設定（空タイルのみ成功）・クリアを検証
  - 既にアイテムを保持するタイルへの設定が拒否されることを検証
  - タイル数・アイテム数のカウントが正しいことを検証
  - 全ベルト座標の取得を検証
  - タイル削除時に保持アイテムが消失することを検証
  - _Requirements: 1.2, 5.4, 7.1, 7.2, 7.3_

- [x] 2.2 BeltGridのデータストア実装
  - Dictionary<Vector2i, BeltTileData>による全ベルトタイルの管理を実装
  - add_tile、remove_tile、get_tile、has_tileの基本操作を実装
  - set_item、clear_itemのアイテム操作を実装（空チェック付き）
  - tile_count、item_count、get_all_positionsのクエリメソッドを実装
  - _Requirements: 1.2, 5.4, 7.1, 7.2, 7.3_

- [x] 3. ベルト接続グラフの構築
- [x] 3.1 接続グラフ再計算の単体テスト作成
  - 東向きベルトの下流が東隣のベルトに正しく接続されることを検証
  - 4方向それぞれの接続パターンを検証
  - 向きと合致しない方向への接続が発生しないことを検証（東向きベルトは北隣に接続しない）
  - L字配置（東→南など方向転換）での接続を検証
  - 下流にベルトが存在しない場合、has_downstreamがfalseになることを検証
  - グリッド範囲外への接続が発生しないことを検証
  - _Requirements: 2.4, 2.6, 5.1, 5.2, 5.3_

- [x] 3.2 接続グラフ再計算ロジックの実装
  - rebuild_connectionsメソッドを実装：全ベルトタイルの下流接続を方向に基づいて計算
  - 各タイルの向き方向の隣接セルにベルトが存在するかチェックし、downstream_posとhas_downstreamを設定
  - CoreGridとPlacementSystemを使用して隣接セルのエンティティ情報を取得
  - _Requirements: 2.4, 2.6, 5.1, 5.2, 5.3_

- [x] 4. ベルト搬送システムの基本処理
- [x] 4.1 アイテム進行の単体テスト作成
  - 1ティックでアイテムの進行度が正しい量（1/60）だけ加算されることを検証
  - 60ティック経過で進行度がほぼ1.0に到達することを検証（1タイル/秒の搬送速度）
  - アイテムを保持していないタイルでは進行度が変化しないことを検証
  - _Requirements: 1.1, 1.3_

- [x] 4.2 (P) アイテム転送の単体テスト作成
  - 進行度1.0到達時に下流ベルトへアイテムが正確に1回転送されることを検証
  - 転送後の元タイルが空（item_id=0, progress=0.0）になることを検証
  - 転送後の先タイルにアイテム（item_id, progress=0.0）が設定されることを検証
  - 転送前後でアイテム総数が保存されることを検証
  - _Requirements: 2.1, 7.1, 7.2, 7.3_

- [x] 4.3 (P) 転送先なし・転送先満杯での待機テスト作成
  - 下流接続なし（has_downstream=false）の場合、進行度1.0で待機することを検証
  - 下流タイルが既にアイテム保持中の場合、進行度1.0で待機することを検証
  - 待機中もアイテムが消失しないことを検証
  - _Requirements: 2.2, 2.3_

- [x] 4.4 搬送システムの基本tickロジック実装
  - BeltTransportSystemのtick()メソッドを実装
  - 1ティックあたりの進行度加算量（SPEED_PER_TICK = 1.0 / 60.0）を定数化
  - 処理順序キャッシュ（末尾→先頭）に従って各ベルトタイルを順次処理
  - 進行度加算 → 1.0到達時に下流転送（受入可能な場合）→ 不可なら進行度1.0でクランプ
  - ダーティフラグチェック：trueならrebuild_connectionsとrebuild_process_orderを実行してからリセット
  - _Requirements: 1.1, 1.3, 2.1, 2.2, 2.3, 7.1, 7.2, 7.3_

- [x] 5. バックプレッシャーの実装
- [x] 5.1 バックプレッシャーの単体テスト作成
  - 出力先満杯でベルト末端アイテムが停止することを検証
  - 末端停止時に後続アイテムも連鎖的に停止することを検証（圧力の逆伝播）
  - 出力先に空きが生じた時点で停止アイテムが自動再開することを検証
  - バックプレッシャー中もアイテム保存則が維持されることを検証
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 5.2 バックプレッシャーロジックの実装
  - 末尾→先頭の処理順序により、下流が移動不可→上流も移動不可の連鎖を自然に実現
  - 出力先に空きが生じた場合の自動再開は、次ティックで下流から処理することで実現
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 6. 処理順序キャッシュの構築
- [x] 6.1 処理順序（末尾→先頭）の単体テスト作成
  - 直線5本ベルトチェーンで末尾（最下流）から先頭（最上流）の順序で処理されることを検証
  - L字配置でも正しい処理順序が構築されることを検証
  - 独立した複数のベルトチェーンがすべて含まれることを検証
  - FIFO: ベルトチェーンに投入した順序でアイテムが出力されることを検証
  - 飛び越え防止: アイテムが占有中の中間ベルトをスキップしないことを検証
  - _Requirements: 6.1, 6.2, 6.3_

- [x] 6.2 処理順序キャッシュの実装
  - rebuild_process_order()メソッドを実装：全ベルトを末尾→先頭（下流→上流）の順に並べたArray[Vector2i]を構築
  - 深さベースソート：各ベルトの「下流からの距離」を計算し、距離が小さい順（末尾が先）にソート
  - 配置/撤去時のみ再構築（ダーティフラグ方式）
  - _Requirements: 6.1, 6.2, 6.3_

- [x] 7. 配置・撤去との統合
- [x] 7.1 配置・撤去通知の単体テスト作成
  - ベルト配置通知（on_entity_placed）でBeltGridにタイルが追加されることを検証
  - ベルト撤去通知（on_entity_removed）でBeltGridからタイルが削除されることを検証
  - 撤去時にベルト上のアイテムが消失することを検証
  - ベルト以外のエンティティ（entity_type_id != 3）の通知が無視されることを検証
  - 配置/撤去後にダーティフラグが設定されることを検証
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 7.2 配置・撤去統合ロジックの実装
  - on_entity_placed: entity_type_idがBelt(ID=3)の場合、BeltGridにタイルを追加しダーティフラグを設定
  - on_entity_removed: entity_type_idがBelt(ID=3)の場合、BeltGridからタイルを削除しダーティフラグを設定
  - ベルト以外のエンティティの通知は無視
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 8. 機械ポートとの入出力
- [x] 8.1 機械ポート入出力の単体テスト作成
  - 機械出力ポートから隣接する空ベルトへアイテムが転送されることを検証
  - ベルト末端から機械入力ポートへアイテムが引き渡されることを検証
  - 機械入力ポート満杯時にバックプレッシャーが適用されることを検証
  - 隣接ベルト満杯時に機械出力ポートからの転送が行われないことを検証
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 8.2 機械ポート入出力ロジックの実装
  - tick()処理内で、機械出力ポートに隣接する空ベルトへのアイテム受入を処理
  - tick()処理内で、ベルト末端から機械入力ポートへのアイテム引き渡しを処理
  - 機械入力ポートが満杯の場合はベルト上にアイテムを保持しバックプレッシャーを適用
  - 注意: 機械ポートの詳細な接続解決は別機能のスコープ。本タスクではインターフェースベースの基本的な受け渡しを実装
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 9. 統合テストとE2Eシナリオ
- [x] 9.1 直線ベルトチェーンのE2Eテスト作成
  - 直線5本ベルトチェーンの一端にアイテムを投入し、他端まで順序を保って到達することを検証
  - 複数アイテムを連続投入し、FIFO順序が維持されることを検証
  - _Requirements: 2.5, 6.1, 6.3_

- [x] 9.2 (P) 曲線配置のE2Eテスト作成

- [x] 9.3 (P) バックプレッシャーE2Eテスト作成

  - アイテム保存則（投入数 = 出力数 + ベルト上残存数）を検証
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 7.1_
